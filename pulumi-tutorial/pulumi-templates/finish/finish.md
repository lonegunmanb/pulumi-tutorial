# 完成

你已经完成 Pulumi Templates 实验。

这次你练习了：

- 查看一个包含 template 段的 Pulumi.yaml。
- 用本地路径执行 pulumi new。
- 使用 name、description、stack 和 config 参数生成项目。
- 验证 PROJECT 与 DESCRIPTION 的变量替换。
- 验证 template config 中的 secret 配置。
- 对生成项目运行 npm install、pulumi preview 和 pulumi up。
- 理解 Git 仓库与子目录 template 的分发方式。

回到正文继续阅读时，可以把 Template 理解成团队项目的标准起点；真正的资源复用仍然应交给 Component 或 Package。
